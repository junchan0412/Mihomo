# Mihomo v1.11.2

`v1.11.2` 聚焦覆写检查、配置来源解释和设置导航一致性。覆写不再依赖系统 Quick Look，设置也不再离开主窗口。

## 覆写快速查看与概览

- 覆写列表继续使用 Space 快捷键，但打开应用自有的 value-based 预览窗口。
- 预览窗口展示名称、类型、来源、状态、作用范围、行数和大小，并提供行号与 YAML/JavaScript 语法高亮。
- 多选覆写后可在预览窗口中切换上一个/下一个项目。
- 覆写页底部由“覆写内容”改为“覆写概览”，汇总语法状态、行数、大小和 YAML 顶层键。
- YAML 使用 Yams 检查语法、顶层映射和重复键；JavaScript 使用 JavaScriptCore 进行纯语法检查，并确认 `transform(config)` 入口。
- Sniffer 的域名、端口和地址规则加入语义检查，问题会显示具体行号。

## 配置质量

- 配置摘要卡现在填满详情区宽度，移除右侧无意义空隙。
- 每条配置质量问题显示来源 badge：当前 Profile、App 设置、覆写或最终配置。
- 域名嗅探端口、域名和地址格式问题明确标注为 App 设置来源，避免误认为 Profile YAML 本身报错。

## 设置与延迟测试

- 设置成为主窗口侧栏中的正式页面；侧栏、菜单栏和 `Command-,` 使用同一导航逻辑。
- 移除独立 Settings window，避免同一设置草稿在多个窗口中出现状态分叉。
- 延迟测试从远程访问移入通用设置。
- 保留原有代理节点测速 URL，并新增独立的 DIRECT 测试 URL。
- DIRECT 通过禁用系统代理的临时 URLSession 发起直连测速；普通代理节点继续通过 mihomo Controller 测速。
- 设置 schema 升级到 v7，旧设置保留代理测速 URL，并自动补入 DIRECT 默认 URL。

## 验证

- 128 项 Swift 测试全部通过，0 失败。
- 新增覆写 YAML/JavaScript 分析、Sniffer 规则行号、设置 v7 迁移和配置质量来源测试。
- 真实 App 验证了 Space 自定义预览、覆写概览、配置摘要宽度、App 设置来源 badge、主窗口设置和 `Command-,`。
- `git diff --check`、可维护性审计、v1.11.2 构建签名、启动、发布包和更新替换 smoke 均纳入发布门禁。
