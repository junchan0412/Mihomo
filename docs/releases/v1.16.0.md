# Mihomo v1.16.0

`v1.16.0` 聚焦实时控制路径与界面刷新的重量级性能重构，降低高频流量/连接更新对 SwiftUI 与 AppKit 表格的冲击，并统一交互动画节奏。

## 实时数据与控制器

- 控制器轮询拆分为元数据（模式/策略组/版本）与连接刷新：WebSocket 健康时仅低频补齐元数据，连接表优先由事件流驱动。
- 连接 WebSocket 采样间隔调整为 750ms，降低主线程与 UI 压力。
- 流量采样继续节流；相同速率在采样窗口内不重复追加图表点。
- 连接结构变化与纯字节变化分流：结构变化立即刷新规则命中统计；字节变化节流发布，避免整表 thrash。
- 活跃连接 ID 使用集合查询，连接/最近请求合并索引减少重复分配。

## 表格与连接工作区

- `AppKitTable` 在行 ID 稳定时改为差分 `reloadData(forRowIndexes:)`，保留滚动位置并减少闪烁。
- 连接页表格行缓存指纹重建，避免每次 body 求值都完整排序与映射。
- 活跃连接判断改为 O(1) 集合查询。

## 交互与动画

- 统一 `MihomoUI.Motion`（quick / snappy / soft）。
- 主窗口分区切换使用更顺滑的软过渡；侧栏选择与连接计数支持 numeric transition。
- 概览指标、菜单栏速率继续使用 monospaced digit 与 numeric content transition。

## 无开发者账户发行说明

本版本使用 ad-hoc 签名，未经过 Apple notarization。下载、解压并移动到 `/Applications` 后，如 macOS 阻止启动，请执行：

```bash
xattr -cr /Applications/Mihomo.app
```

首次启动核心或修复 Helper 时，macOS 会要求管理员授权。应用内更新继续通过 Ed25519 manifest、zip SHA-256、bundle identifier，以及主 App 与 Helper 的精确 CDHash 验证产物。
