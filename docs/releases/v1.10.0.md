# Mihomo v1.10.0

`v1.10.0` 重构了客户端与 mihomo 核心之间的控制通道，并把 Sniffer 转化为普通用户可以理解和操作的“域名嗅探”。

## 控制通道与远程管理

- Mihomo 自动管理本机 HTTP/WebSocket 控制通道，客户端始终通过 `127.0.0.1` 访问自己启动的核心。
- 最终 Runtime Config 会在 Profile、JS Transform 和 YAML 覆写之后固定写入 `external-controller` 与 `secret`，并移除 Unix/Pipe endpoint，避免配置覆盖导致客户端失联。
- 设置页不再暴露“本机 Controller”，改为“远程管理”“监听地址”“管理端口”和“访问密钥”等用户语义。
- 只有开启“允许其他设备管理此 Mac”时才展开远程参数；回环地址自动切换为 `0.0.0.0`，空密钥自动生成。
- “允许局域网设备使用代理”与远程管理权限拆分，使用代理不再隐含获得核心管理能力。

## 域名嗅探

- Sniffer 统一命名为“域名嗅探”，并说明它从 HTTP Host、TLS SNI 和 QUIC 握手识别域名，不解密 HTTPS，也不是 DNS 查询。
- 网络工作区新增“域名嗅探”分段，概览页提供状态、快速开关和“详细设置”入口。
- 支持分别配置 HTTP、TLS、QUIC 端口，以及 `parse-pure-ip`、`force-dns-mapping` 和 `override-destination`。
- 支持强制/跳过域名、跳过来源地址和目标地址，地址可使用 IP 或 CIDR。
- 默认由 Mihomo 管理最终 `sniffer`；高级用户关闭接管后，配置完全遵循 Profile、JS Transform 与 YAML 覆写。

## 迁移与可靠性

- 设置 schema 升级到 v5，自动迁移旧 Sniffer 端口、归一化本机控制地址，并为已有远程管理配置补齐访问密钥。
- 客户端请求、策略测速、诊断、备份和 WebSocket stream 统一使用应用管理的本机控制地址。
- 修复关闭主窗口后“显示主窗口”无法重新打开窗口的问题。
- 更新 README、开发报告、辅助功能检查清单与控制通道/域名嗅探设计文档。
