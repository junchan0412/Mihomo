# Mihomo v1.11.0

`v1.11.0` 重构网络概览、清理不属于客户端职责的 External UI 管理，并建立 Profile 与 App 常用设置之间的双向同步。

## 网络界面

- 系统代理、TUN / 路由和系统 DNS 从三张大卡改为紧凑行式控制区，减少空白和指针移动距离。
- 每一行同时显示用途、实际状态和开关，并补齐 accessibility label、value 与 hint。
- 域名嗅探从接管方式中独立为“流量识别”，明确它不会改变系统接管范围。
- 域名嗅探详情页改为与当前 Profile 同步，不再显示额外的“由 Mihomo 管理”所有权开关。

## Geo 数据

- 删除 External UI 下载、安装、状态、诊断和 Runtime Config 注入。
- 默认 Geo 数据补齐为 `geoip.dat`、`geosite.dat`、`Country.mmdb` 和 `ASN.mmdb`。
- 默认 URL 使用 MetaCubeX `meta-rules-dat` 最新 Release；手动 SHA-256 留空时自动读取对应 `.sha256sum`。
- 四项文件下载后原子替换，并同步复制到 Runtime 目录。
- 真实下载验证四项资产均为 HTTP 200，下载文件 SHA-256 与上游校验值一致。

## Profile 与 App 设置同步

- 启用、导入或刷新 Profile 时，将端口、LAN、日志等级、TUN、DNS 和域名嗅探值载入 App。
- 修改这些 App 设置并应用时，只把发生变化的字段写回当前 Profile。
- 保留节点、策略组、规则、Provider 和 App 不管理的配置内容。
- App-only 设置不会触发 Profile 重写；远程订阅刷新后，远端内容重新覆盖本地缓存并载入 App。
- Runtime Config 仍保持 `App 默认 < Profile < JS Transform < YAML 覆写`，控制通道是唯一最终强制写入的例外。
