# Mihomo v1.13.0

`v1.13.0` 深度重构系统代理、TUN、DNS、域名嗅探与连接请求，并完善无 Apple Developer 账户时的 Helper 和更新链路。

## 核心网络能力

- TUN 开启时强制启用 Mihomo DNS，并组合 Fake-IP、`dns-hijack: any:53`、自动路由与接口探测，避免只改写 macOS DNS 却未接管实际解析流量。
- 系统代理、系统 DNS 与 TUN 使用互相独立的快照；系统代理可以与 TUN 协同启用，停止核心、退出 App 和更新前都会按实际接管状态分别恢复。
- Sniffer 支持 HTTP、TLS、QUIC、纯 IP 解析、DNS mapping、目标替换、强制/跳过域名以及源/目标地址例外。
- 连接事件使用 `/connections?interval=500`，忽略不完整状态帧；活动连接结束后稳定进入去重历史，详情优先展示 `sniffHost`。
- 连接界面只保留最近请求、活动连接、DNS 与流量统计；删除重复的设备和日志模块，日志使用独立重构页面。

## Helper、更新与恢复

- Helper XPC 调用增加有界超时和可执行的恢复提示。
- 无 Apple Team 签名时跳过无效的 SMAppService 批准等待，改用需管理员明确授权的传统 Helper。
- 传统 Helper、LaunchDaemon 和授权文件均由 root 持有；授权绑定 `/Applications/Mihomo.app`、bundle identifier 与精确 CDHash。
- 软件更新先验证候选 App，再停止核心并恢复 Proxy/DNS/TUN，随后停用旧 Helper；替换失败会恢复旧 App 与 Helper 状态。
- `CFBundleVersion` 改为数字 build，不再使用 Git 短哈希。

## 无开发者账户发行说明

本版本为 ad-hoc 签名，未经过 Apple notarization。下载并移动到 `/Applications` 后，如 macOS 阻止启动，请执行：

```bash
xattr -cr /Applications/Mihomo.app
```

首次启动核心或点击“修复 Helper”时，macOS 会要求管理员授权以安装传统 Helper。应用内更新由 Ed25519 manifest、zip SHA-256、bundle identifier，以及主 App 与 Helper 的精确 CDHash 共同验证。

## 验证

- 147 项 XCTest 全部通过。
- CI release gate 与 GitHub Actions 通过。
- 真实传统 Helper 安装验证：root ownership、LaunchDaemon、XPC 版本、App 路径/build 绑定与 root 权限均通过。
- 实际 Sniffer/Connections 数据面验证：SOCKS 目标为纯 IP、TLS SNI 为 `example.com` 时，Controller 返回 `host` 与 `sniffHost=example.com`，并按域名规则选择代理链。

实现参考 [xishang0128/sparkle](https://github.com/xishang0128/sparkle) 的 DNS/TUN、Sniffer 与 Connections 设计，同时保留 Mihomo macOS 的独立恢复快照、Profile↔App 设置同步、连接历史与策略流量统计。
