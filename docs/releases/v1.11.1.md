# Mihomo v1.11.1

`v1.11.1` 将覆写工作区重构为与配置页一致的 macOS 管理界面，并为 YAML 与 JavaScript 覆写补齐本地和远程来源生命周期。

## 覆写界面

- 页面结构与配置页保持一致：标题、存储位置、订阅状态、表格、操作区、摘要和内容预览。
- 表格显示应用顺序、启用状态、名称、类型、来源、作用范围和更新时间。
- 支持多选启停、删除、Quick Look、复制内容以及通过右键菜单调整应用顺序。
- 双击、Return 或“编辑”会打开独立覆写编辑器窗口；新建覆写也使用相同编辑器。
- 编辑器支持 YAML / JavaScript、全局 / 指定 Profile、内容校验、Undo/Redo 和远程内容覆盖提示。

## 导入与订阅

- 支持导入 `.yaml`、`.yml`、`.js` 和普通文本文件，也可直接拖入覆写页。
- 支持从 HTTP / HTTPS URL 导入 YAML 或 JavaScript 覆写。
- 远程覆写保存来源 URL 和 TLS 证书指纹，可刷新单项或全部远程覆写。
- 菜单栏、App Intent 和自动订阅刷新会同时刷新远程 Profile 与远程覆写。
- `mihomo://install-fragment?...&url=...` 深链导入的覆写会保留远程来源，因此后续可以刷新。
- 支持将覆写导出为 YAML / JavaScript 文件。

## 兼容与安全

- 旧版 `config-fragments.json` 缺少来源字段时自动按本地覆写读取。
- URL 仅接受 HTTP / HTTPS，响应必须为 2xx 和有效 UTF-8 文本。
- YAML 必须是顶层映射且不超过 2 MiB；JavaScript 保持 64 KiB 上限。
- 远程刷新继续校验证书指纹，避免订阅源证书意外变化后静默替换内容。
