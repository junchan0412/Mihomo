# 桌面客户端差距分析

更新日期：2026-07-24。

本文把外部产品的公开资料与当前 Mihomo 的实现分开记录。竞品功能是资料事实；优先级与路线是本项目的判断，不代表竞品优劣排名。

## 已完成的基线

- 策略页与菜单栏可显示节点延迟、按延迟排序和筛选，并可按组或全量测速。
- 节点提供商可批量导入、分组、打标签，并与 Profile 的 `proxy-providers` 双向同步。
- Profile 远程刷新保留上次内容中缺失的 Provider，避免上游响应遗漏破坏本地关联。
- 侧栏支持可折叠分区、快捷收藏和统一背景。

## 已完成的 P0：数据安全与可解释性

1. Provider 同步改为 `proxy-providers` 段内的局部文本编辑，仅变动受管理字段；顶层顺序、未知字段、块内注释、行内注释与既有缩进保持不变。新建 Provider 会沿用当前段落的缩进。内联 `proxy-providers` 映射会拒绝写入并提示改用标准缩进映射，避免退回全量序列化。
2. Provider 使用标准化名称作为全局唯一键，并保留来源 Profile UUID 作为优先级信息；导入 Profile 时，同名独立 Provider 会自动合并，以 Profile 声明的定义为准，同时保留用户分组、标签与关联。同名记录不再能同时出现在资源列表或阻塞写入预览。
3. 批量导入、编辑、关联切换和删除均先显示 Profile 写入摘要，再确认落盘。应用前会检测 Profile 是否在预览后被修改；写入失败会恢复快照，并提供当前会话的一步撤销。远程刷新在需要保留上游遗漏 Provider 时进入确认队列，逐项确认，不会因批量刷新丢失后续预览。

### P1：高频操作效率

1. 菜单栏增加最近切换节点、收藏策略组和可选快捷键；搜索结果按当前选择、可用性和延迟排序。
2. 为策略组提供延迟历史、失败原因和最近测速时间，而不仅是单次数字。
3. 增加命令面板，统一页面跳转、配置切换、测速、刷新资源和开关网络能力。
4. 资源页支持基于分组和标签的多选批量启停、刷新、迁移和导出。

### P2：规则与系统体验

1. 增加按进程、域名、规则集和策略组串联的路由解释视图，说明某次连接为何命中某条规则。
2. 增加 Profile/覆写的结构化 diff、版本快照和按字段回滚。
3. 提供可导出的隐私化运行健康报告，帮助排查 TUN、DNS、系统代理和资源更新问题。

## 自检结论

Provider 写回主路径已经具备格式保真、来源去歧义、冲突解释和可逆操作。文本编辑器的边界仍是标准缩进 mapping：对于内联映射会明确拒绝写入，避免以不安全的全量序列化替代。下一轮可按 P1 提升菜单栏的个性化与延迟历史。
